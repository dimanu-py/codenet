# Design Documents

This directory contains design documents for features and architectural decisions in the Codenet project.

## Purpose

Design docs capture ideas, requirements, and implementation plans for future features. They serve as:

- Planning artifacts before implementation
- Documentation of architectural decisions
- Reference for understanding feature context and requirements

## How to Create a Design Doc

1. **Copy the template**: Use `docs/templates/design_docs/feature_template.md`
2. **Name the file**: Use format `NNNN-kebab-case-title.md` where NNNN is the next sequential number (e.g., `0001-feature-name.md`)
3. **Fill metadata**: Set `created` date (YYYY-MM-DD) and `status` value
4. **Write concisely**: Use bullet points, code examples, and clear structure. Avoid verbose prose.
5. **Update this index**: Add entry to the appropriate status table below

## Status Values

- **Proposed**: Idea documented, not yet approved or started
- **In Progress**: Currently being implemented
- **Implemented**: Feature completed and merged
- **On Hold**: Paused or waiting for dependencies
- **Rejected**: Decided not to implement

## Numbering Rules

- Start at 0001
- Increment sequentially
- Numbers are permanent (never reused or changed)
- Skip numbers if a design doc is deleted
